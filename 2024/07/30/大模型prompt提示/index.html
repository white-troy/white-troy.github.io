LLM prompt

在自然语言处理（NLP）和生成式预训练模型（如GPT-3和GPT-4）中，”prompt” 是指输入给模型的一段文本或指令，用于引导模型生成相应的输出。Prompt 可以是一个问题、一个不完整的句子、一段描述，或者任何其他形式的文本，旨在激发模型生成相关的、连贯的和有意义的响应。

在语言模型中，prompt的作用类似于一个启动器，通过提供上下文或指令，帮助模型理解用户的意图并生成相应的文本。Prompt 的设计和质量直接影响模型生成的结果。

如何设计prompt

prompt设计的技巧

在编写prompt给大模型时，根据openAI的提示词工程 需要注意以下几点：

  • 写出清晰的指令
  • 提供参考文本
  • 将复杂任务拆分为简单的子任务
  • 给模型时间思考
  • 使用外部工具
  • 系统地测试变更

此外，在编写时可以考虑将prompt进行结构化编写，将prompt作为一个模板，类似于分点列出“是什么”，“为什么”，“怎么做”的过程。如下面的几个prompt框架

ICIO

在 Prompt 工程中，ICIO 框架是一种简洁 Prompt 的框架。ICIO 是以下四个部分的缩写：

  • Instruction（指令）：明确告诉模型你希望它做什么。这部分应该清晰、具体，避免模糊不清的描述。例如：“请你总结下方的这段文字”
  • Context（上下文）：提供必要的背景信息或上下文，帮助模型更好地理解任务。例如：“下面是一份关于气候变化的科学文章”
  • Input data（输入数据）：提供模型需要处理的具体数据或示例。
  • Output indicator（输出指示）：明确指出你希望模型生成的输出格式或风格。例如：“请你按照下面这个格式输出”

BROKE

BROKE 框架融合了 OKR（Objectives and Key Results）方法论，通过使用结构化的方法来提高LLM的输出质量。BROKE主要由下面的几个部分组成

  • 背景 (Background): 提供足够的背景信息，使LLM能够理解问题的上下文。例如：“这是一篇关于气候变化的科学文章”
  • 角色 (Role): 设定特定的角色，让LLM能够根据该角色来生成响应。例如：“你是一位专业的气候科学专家”
  • 目标 (Objectives): 明确任务目标，让LLM清楚知道需要实现什么。例如：“请总结以下文章的主要观点”
  • 知识 (Knowledge): 提供给LLM相关的附加知识
  • 演变 (Evolve): 通过试验和调整来测试结果，并根据需要进行优化。

CRISPE

CRISPE是GPTprompt使用 中一个常用的prompt结构，整体结构具有明确的目的。它主要由以下几个部分构成：

  • 能力和角色 (Capacity and Role): 定义 ChatGPT 应扮演的角色或角色。
  • 洞察 (Insight): 提供有关请求的幕后洞察、背景和上下文。
  • 声明 (Statement): 说明您要求 ChatGPT 完成的任务。
  • 个性 (Personality): 定义您希望 ChatGPT 回应时采用的风格、个性或方式。
  • 实验 (Experiment): 要求 ChatGPT 为您提供多个示例。

下面是一个CRISPE的官方示例
|Step |Example Prompt|
|—-|—-|
|Capacity and Role（能力和角色） |作为软件开发方面的专家，讨论机器学习框架的主题，并作为专家博客作者|
|Insight（洞察） |此博客的受众是对学习最新机器学习进展感兴趣的技术专业人员。|
|Statement（声明） |提供对最流行的机器学习框架的全面概述，包括它们的优势和劣势。包括真实的示例和案例研究，以说明这些框架如何在各种行业中成功应用|
|Personality（个性化） |在回应时，请混合使用 Andrej Karpathy, Francois Chollet, Jeremy Howard 和 Yann LeCun 的写作风格。|
|Experiment（实验） |给我多个不同的示例。|

总结

上面提及的只是一些比较基础的prompt，在引导LLM进行回答时，个人感觉更重要的是给予模型更多的参考信息，无论是期望他给出的输出结果，还是提供给他的背景知识，信息越详细效果会越好。在提供背景知识这里，可以引申出RAG的功能。但如何提升RAG召回的准确率和查全率，也是一个值得深入学习的话题。

在实际实践中，曾遇上过长的Prompt使模型无法准确遵循指令的情况。如在使用llm生成部分数据集时，规定的prompt过长导致prompt中的有效指令部分被稀释，同时提供的context位置与较重要的规则距离较远，在生成的数据集中总有一部分没有跟随指令，导致数据质量降低，提升人工清洗的困难度。

如何在合适的长度与合适的位置给予模型指令与参考信息，也是一个十分需要学习的方向
