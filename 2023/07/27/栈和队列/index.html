栈和队列的行为特征

C++中的栈（Stack）和队列（Queue）都是容器适配器，它们是基于其他容器（如数组、向量、链表等）实现的数据结构，提供了特定的操作接口来满足栈和队列的特性。栈和队列与数组或 list 极其相似，但对插入、访问和删除元素的方式有一定的限制。可将元素插入到什么位置以及可从什么位置删除元素决定了容器的行为特征。

栈

栈是一种后进先出（Last In First Out，LIFO）的数据结构，只能从栈顶插入或删除元素。类似于现实生活中的堆栈，最后放入的元素最先被取出。可以通过std::stack模拟这种行为，使用时要包含头文件< stack >

在栈中，元素只能在栈顶进行插入（压栈，push）和删除（弹出，pop）操作，不允许在其他位置进行插入和删除。时间复杂度为 O(1)。

栈通常用于解决与函数调用、递归和表达式求值相关的问题，如函数调用栈、括号匹配、逆波兰表达式等。

队列

队列是一种先进先出（First In First Out，FIFO）的数据结构，元素被插入到队尾，最先插入的元素最先删除，类似于现实生活中的排队。可以使用std::queue模拟这种行为，使用时要包含头文件< queue >

在队列中，元素只能在队尾进行插入（入队，enqueue）操作，在队头进行删除（出队，dequeue）操作，不允许在其他位置进行插入和删除，时间复杂度为 O(1)。

队列通常用于解决与排队、任务调度等相关的问题，如多线程任务调度、网络请求处理等

使用 STL stack 类

std::stack 是一个容器适配器（container adapter），它是基于其他容器（默认使用 std::deque）实现的栈数据结构。std::stack 提供了栈（后进先出，LIFO）的操作接口，允许用户在栈顶进行元素的插入和删除，但不允许在其他位置进行访问、插入或删除操作。

使用 STL 优先级队列

std::priority_queue 是一个容器适配器（container adapter），用于实现优先队列（priority queue）数据结构。优先队列是一种特殊的队列，它允许在队列中插入元素，并保证每次访问或删除元素时都能获取最高优先级（最大或最小值，取决于优先级的定义）的元素。

std::priority_queue 使用堆（heap）数据结构来实现优先队列，其中较高优先级的元素（或通过二元谓词决定的最大值）会在队列头部（且只能在头部进行操作），较低优先级的元素会在队列尾部。默认情况下，std::priority_queue 使用 std::vector 作为其底层容器，并通过默认的比较函数 std::less 维护元素的降序（最大堆），也可以通过模板参数自定义比较函数来实现升序（最小堆）。使用时要包含头文件< queue >

std::cout << "Priority Queue elements: ";

    pq.pop();
