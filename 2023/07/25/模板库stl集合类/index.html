STL集合类，set和multiset

std::set 和 std::multiset 是 C++ 标准模板库（STL）中的有序关联容器，它们用于存储一组唯一的元素，并根据元素的值自动进行排序。使用时，要包含头文件< set >

std::set 是一个集合容器，它包含一组唯一的元素，且元素按照升序进行排序。它的插入操作在 O(log n) 时间内完成，查找、删除操作也在 O(log n) 时间内完成，其中 n 是容器中元素的数量。插入元素时，如果容器中已经存在相同的元素，则插入操作会失败，因为 set 不允许存储重复的元素。

std::multiset 也是一个集合容器，它与 std::set 类似，但允许存储重复的元素。它的插入操作在 O(log n) 时间内完成，查找、删除操作也在 O(log n) 时间内完成，其中 n 是容器中元素的数量。
multiset 会保留相同值的元素，所以可以存储多个相同的元素。

std::set和std::unorder_set

std::set 和 std::unordered_set 都是 C++ 标准库中的关联容器，它们用于存储一组唯一的元素，并支持高效的查找、插入和删除操作。它们之间的主要区别在于底层的数据结构和元素的排序方式。

  1. 底层数据结构：

    std::set 使用红黑树（Red-Black Tree）作为底层数据结构。红黑树是一种自平衡二叉搜索树，它保持元素有序，并支持对数时间复杂度的查找、插入和删除操作。

    std::unordered_set 使用哈希表（Hash Table）作为底层数据结构。哈希表通过将元素映射到桶中，并使用哈希函数来实现快速查找、插入和删除操作，平均情况下具有常数时间复杂度。

  2. 元素的排序方式：

    std::set 中的元素总是按照元素值进行升序排序。这意味着在 std::set 中存储的元素是有序的，并且可以使用 begin() 和 end() 迭代器来访问有序序列。

    std::unordered_set 中的元素没有特定的排序顺序，它们是根据哈希函数的结果存储在桶中的。因此，std::unordered_set 中的元素没有明确的顺序，不能通过迭代器以特定的顺序访问。

  3. 性能：

    在大多数情况下，std::unordered_set 的查找、插入和删除操作比 std::set 快，因为哈希表提供了常数时间的平均查找时间。
    然而，在特定的情况下，std::set 可能比 std::unordered_set 更高效，尤其是在元素数量较小或者有序访问的场景下。

  4. 空间复杂度：

    std::set 的空间复杂度受到红黑树的影响，通常比 std::unordered_set 更高，因为红黑树需要维护额外的指针和颜色信息。

    std::unordered_set 的空间复杂度取决于哈希表的桶的数量和元素数量，通常比 std::set 更低，尤其是当哈希表的负载因子较小时。

如果需要有序存储并支持快速查找、插入和删除操作，并且对于空间效率没有特别高的要求，则可以选择 std::set。如果对于查找、插入和删除操作的性能有更高的要求，并且对于元素的顺序没有特别的要求，则可以选择 std::unordered_set。
