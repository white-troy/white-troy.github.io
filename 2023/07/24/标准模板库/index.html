标准模板库简介

C++的标准模板库（Standard Template Library，STL）是C++标准库的一个重要组成部分，它为开发人员提供了一组通用的模板类和函数，用于处理常见的数据结构和算法。STL是C++标准的核心部分，包含三个主要组件：容器，算法和迭代器

STL容器

STL提供了多种容器类，用于存储和管理数据。常见的容器有：vector（动态数组）、list（双向链表）、deque（双端队列）、set（集合）、map（映射表）等。STL提供了顺序容器和关联容器两种类型，这两种容器用于不同的数据储存和访问方式。另外，STL还提供了容器适配器的类，是顺序容器和关联容器的变种，包含的功能有限，用于满足特殊的需求。

概念补充：复杂度

复杂度用于衡量算法或数据结构的性能和效率，主要分为时间复杂度和空间复杂度两种,好的算法应该具备时间效率高和存储量低的特点

时间复杂度

空间复杂度

空间复杂度衡量算法在执行时所需的额外内存空间。通常使用大 O 表示法来表示空间复杂度，表示算法所需的额外空间与输入规模 n 的增长率之间的关系。

空间复杂度与时间复杂度类似，也有常数空间复杂度 O(1) 和线性空间复杂度 O(n) 等。在使用空间时，需要注意算法是否创建了新的数据结构或临时变量来存储数据，这些都会影响空间复杂度。

顺序容器

顺序容器是一种线性存储数据的容器（如数组和链表），元素的排列顺序与它们被插入的顺序一致。这些容器允许通过位置或迭代器来访问元素，可以随机访问其中的元素，具有插入速度快但查找相对较慢，适用于需要频繁随机访问的场景。

常见的顺序容器有：

  1. std::vector：操作与动态数组一样，支持随机访问和动态增减元素
  2. std::list: 操作与双向链表类似，内部使用双向链表的数据结构来存储数据，支持快速的插入和删除操作，但不支持随机访问
  3. std::deque: 双端队列，支持随机访问和在头尾进行快速的插入和删除操作，但不支持随机访问
  4. std::forward_list: 类似于std::list，但是是一个单向的链表，只能沿一个方向遍历
  5. 补充： 在链表中，元素之间的相对位置是由指针来维护的，而不是通过元素的索引。这意味着链表不支持直接的随机访问，即不能像数组或 std::vector 那样通过索引快速访问元素。如果需要访问链表中的第N个元素，需要从链表头或尾开始逐个遍历节点，直到达到目标位置。因此，链表的访问时间复杂度是O(N)，其中N是目标位置的索引。

关联容器

关联容器是一种基于键（Key）的数据结构，每个元素都有一个唯一的键值，根据这个键值可以快速检索和访问元素。关联容器使用树结构（如红黑树）来组织数据，因此对于大规模数据的查找操作具有较好的性能。关联容器中的元素是按照键值有序存储的。

常见的关联容器有：

  1. std::set：存储各不相同的值，在插入时进行排序；容器的复杂度为对数。无重复元素的集合，元素按照键值对有序存储
  2. std::unordered_set：存储各不相同的值，在插入时进行排序；容器的复杂度为常数。这种容器是 C++11 新增的。
  3. std::map：存储键-值对，并根据唯一的键排序（按照键值有序存储）；容器的复杂度为对数。
  4. std::unordered_map：存储键-值对，并根据唯一的键排序；容器的复杂度为对数。这种容器是
    C++11 新增的。
  5. std::multiset：与 set 类似，但允许存储多个值相同的项，即值不需要是唯一的（允许重复元素）。元素按照键值对有序存储
  6. std::unordered_multiset：与 unordered_set 类似，但允许存储多个值相同的项，即值不需要是唯一的。这种容器是 C++11 新增的。
  7. std::multimap：与 map 类似，但不要求键是唯一的。按照键值有序存储
  8. std::unordered_multimap：与 unordered_map 类似，但不要求键是唯一的。这种容器是 C++11新增的。

容器适配器

容器适配器（Container Adapters）是C++标准模板库（STL）提供的一种特殊类型的容器，它们是通过对其他STL容器进行封装和适配而创建的。容器适配器允许使用不同的接口来访问已有容器的功能，从而方便地实现特定功能或实现不同的数据结构。主要有以下三种常见的适配器：

  1. std::stack：std::stack 是一个基于某个已有容器的适配器，它提供了栈（stack）的功能，即先进后出（Last-In-First-Out，LIFO）的数据结构，亦可在栈顶插入和删除元素。默认情况下，std::stack 使用 std::deque 作为其底层容器，但也可以通过指定其他容器类型来实现不同的底层数据结构。
  2. std::queue：std::queue 是另一个基于已有容器的适配器，它提供了队列（queue）的功能，即先进先出（First-In-First-Out，FIFO）的数据结构，可以删除最先插入的元素。默认情况下，std::queue 使用 std::deque 作为其底层容器，但同样也可以通过指定其他容器类型来实现不同的底层数据结构。
  3. std::priority_queue：std::priority_queue 是一个基于已有容器的适配器，它提供了优先队列（priority queue）的功能，即能够按照元素的优先级进行插入和弹出。默认情况下，std::priority_queue 使用 std::vector 作为其底层容器，但同样也可以通过指定其他容器类型来实现不同的底层数据结构。

STL迭代器

STL迭代器（Iterator）是一种通用的接口，用于在STL容器中遍历和访问元素，而无需关心容器的具体类型和实现细节。迭代器提供了一种统一的方法来访问容器中的元素，使得算法可以在不关心容器类型的情况下工作，从而增加了代码的灵活性和可重用性。

STL中的每个容器都有自己的迭代器类型，它们的特性和行为可能会有所不同，但在使用时，它们遵循相同的接口规范。迭代器通常支持以下操作：

  1. 解引用（Dereferencing）：通过 * 运算符获取迭代器所指向的元素的值
  2. 移动（Advancing）：通过 ++ 运算符将迭代器指向下一个元素。
  3. 比较（Comparing）：通过 ==、!= 等运算符比较两个迭代器是否相等。
  4. 访问容器成员函数：某些迭代器还可以通过特定的成员函数访问容器的元素，例如 -> 运算符用于访问指向对象的成员。

STL 中的迭代器是模板类，从某种程度上说，它们是泛型指针，最简单的迭代器是指针，给定一个指向数组中的第一个元素的指针，可递增该指针使其指向下一个元素，还可直接对当前位置的元素进行操作。STL提供的迭代器可以分为输入迭代器和输出迭代器两种。

  1. 输入迭代器：通过对输入迭代器解除引用，它将引用对象，而对象可能位于集合中。最严格的输入迭代器确保只能以只读的方式访问对象。
  2. 输出迭代器：输出迭代器让程序员对集合执行写入操作。最严格的输出迭代器确保只能执行写入操作。

这两种基本迭代器可以进一步分为前项迭代器，双徐昂迭代器和随机访问迭代器这三类：

  1. 前向迭代器：这是输入迭代器和输出迭代器的一种细化，它允许输入与输出。前向迭代器可以是 const 的，只能读取它指向的对象；也可以改变对象，即可读写对象。前向迭代器通常用于单向链表。
  2. 双向迭代器：这是前向迭代器的一种细化（可以视为继承或者具体化），可对其执行递减操作，从而向后移动。双向迭代器通常用于双向链表。
  3. 随机访问迭代器：这是对双向迭代器的一种细化，可将其加减一个偏移量，还可将两个迭代器相减以得到集合中两个元素的相对距离。随机访问迭代器通常用于数组。

STL算法

STL（标准模板库）提供了丰富的基本算法，这些算法是对通用的数据结构和容器的操作。这些算法是通过迭代器来工作的，使得它们能够在不同类型的容器上通用使用，从而提高了代码的灵活性和可重用性。以下是一些常见的STL基本算法：

  1. std::find(): 在容器中查找指定值的元素，返回一个指向第一个匹配元素的迭代器，如果未找到，则返回末尾迭代器。
  2. std::sort(): 对容器中的元素进行排序，默认按照升序进行排序，可以通过自定义比较函数来实现其他排序方式。
  3. std::reverse(): 将容器中的元素逆序排列。
  4. std::copy(): 将一个容器中的元素复制到另一个容器中。
  5. std::max_element() 和 std::min_element(): 分别查找容器中的最大和最小元素，返回指向它们的迭代器。
  6. std::count(): 计算容器中指定值的元素个数。
  7. std::accumulate(): 对容器中的元素进行累积操作，例如求和。
  8. std::for_each(): 对容器中的每个元素执行指定操作。
  9. std::transform(): 对容器中的元素进行转换，并将结果存储到另一个容器中。
  10. std::remove(): 在容器中删除指定值的元素，返回一个指向新的结尾的迭代器。
  11. std::unique(): 在容器中移除相邻的重复元素，返回一个指向新的结尾的迭代器。
