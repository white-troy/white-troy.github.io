宏

在C++中，宏是一种预处理指令，用于在编译代码之前进行文本替换。它是C++预处理器的一部分，用于在编译阶段对源代码进行简单的文本替换，将宏定义的标识符替换为相应的文本。

宏可以用于定义常量，如可以在程序开始时定义一个常量#define PI 3.14，这样在下面遇到标识符PI时，程序就会认识到这是3.14(宏仅仅是简单的文本替换，没有进行类型检查，所以要设置正确的参数类型)

宏也可以用于定义函数，通过宏，可以创建简单的代码片段，类似于函数的功能。当宏被调用时，预处理器会将宏名替换为相应的代码块，因此，宏调用的效率比函数调用高，但它不能像函数那样拥有独立的作用域和类型检查。此外，宏也可以允许在多个地方使用相同的代码片段，提高代码的重用性

在有预处理指令（如#ifdef, #ifndef, #else, #endif等）的地方，可以根据不同的定义使用不同的代码块，从而实现在不同平台下编译不同的代码（如#ifdef WINDOWS #ifdef LINUX）

使用#define 编写宏函数

使用 #define 预处理指令定义宏函数，指定宏名和宏体（替换文本）。宏体可以是任意有效的C++代码片段，包括表达式、语句、甚至是其他宏。宏函数的优点是在预处理阶段进行替换，避免了函数调用的开销，从而提高了执行效率。但宏函数也有一些缺点和注意事项：

  1. 宏函数是简单的文本替换，没有类型检查和作用域，可能导致不符合预期的结果
  2. 宏函数不会进行代码块的花括号 {} 包裹
  3. 使用宏函数时需要注意表达式的正确性和括号的使用，以避免优先级和运算顺序带来的问题，这也是在定义宏函数中使用大量括号的原因，如下面这个定义计算圆面积的宏函数：

模板

C++的模板（Templates）是一种泛型编程的特性，它允许编写通用的代码，可以用于处理多种不同的数据类型，而无需为每种数据类型编写重复的代码。模板在C++中被广泛应用于容器类（例如STL中的vector、list等）、算法（例如STL中的sort、find等）、函数和类的设计等场景，使得代码更加灵活和可重用。

模板的工作原理是在编译时根据模板参数生成特定类型的函数或类的实例化代码。在实际使用中，通过传递不同的数据类型作为模板参数，编译器会根据模板参数生成对应类型的具体代码。

在C++中有两种类型的模板：

  1. 函数模板（Function Templates）：用于定义通用的函数，可以处理多种类型的参数。函数模板通过使用template关键字和模板参数来定义。
  2. 类模板（Class Templates）：用于定义通用的类，可以适用于多种数据类型。类模板通过使用template关键字和模板参数来定义。
    无论是哪种类型，都是以关键字template打头，然后是类型参数列表。

各种类型的模板声明

模板声明可以是：

  1. 函数的声明或定义；
  2. 类的定义或声明；
  3. 类模板的成员函数或成员类的声明或定义；
  4. 类模板的静态数据成员的定义；
  5. 嵌套在类模板中的类的静态数据成员的定义；
  6. 类或类模板的成员模板的定义

参数数量可变的模板

C++11引入了参数数量可变的模板（Variadic Templates）特性，允许在模板中接受可变数量的参数，从而使模板更加灵活和通用。

参数数量可变的模板的基本语法是使用三个点号’…’ 来表示参数包（Parameter Pack）。在模板参数列表中，可以将一个参数包（即一个参数的集合）放在末尾位置，并用’…’表示。然后在模板定义中，可以使用 sizeof’…’ 操作符获取参数包中的参数数量，以及通过展开参数包的方式处理其中的参数。下面是一个示例：


在上述例子中，程序定义了一个可变参数函数模板 ‘print’。函数模板有两个版本：一个是递归终止版本，用于处理不带参数的情况，另一个是可变参数版本（接受一个 T 类型的参数 value 和一个参数包 Args… args。），用于处理带参数的情况。

在 print 函数模板的实现中，程序首先输出当前参数 value，然后通过递归调用 print(args…) 处理剩余的参数，同时使用 sizeof… 操作符输出剩余参数包中的参数数量。由于参数数量可变，递归调用会不断展开参数包，直到参数包为空，即调用了递归终止版本，结束递归。

在 main 函数中，调用 print 函数模板并传入不同类型和数量的参数。编译器会根据实际调用的参数推导模板参数类型，并生成相应的函数实例。

使用 static_assert 执行编译阶段检查

与assert宏类似，在调试阶段可以通过assert对程序进行检查，在不满足指定条件时禁止编译。如要禁止针对类型 int 实例化模板类，可使用 static_assert( )，并将 sizeof(T)与 sizeof(int)进行比较，
如果它们相等，就显示一条错误消息：
static_assert(sizeof(T) != sizeof(int), “No int please!”);
