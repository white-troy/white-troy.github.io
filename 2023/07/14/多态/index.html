多态

什么是多态

1.多态可以以一种统一的方式处理不同类型的对象，并在运行时确定要调用的实际函数，主要形式有静态多态和动态多态两种。

1.1 静态多态主要通过函数重载和模板来实现。通过函数重载传入不同的参数，从而来确定调用哪个函数
1.2 动态多态通过继承与虚函数来实现。在基类中，可以将函数声明为虚函数，允许派生类重新定义该函数以适应自身的特定需求。通过基类的指针或引用调用虚函数时，会在运行时根据对象的实际类型来确定要调用的函数

使用虚函数实现多态

1.使用关键字virtual将函数声明为虚函数，确保编译器调用的是覆盖版本

2.对于使用 new 在自由存储区中实例化的派生类对象，如果将其赋给基类指针，并通过该指针调用 delete，将不会调用派生类的析构函数。这可能导致资源未释放、内存泄露等问题，此时可将析构函数声明为虚函数，确保通过基类指针调用 delete 时，将调用派生类的析构函数

抽象基类和纯虚函数

1.抽象基类（Abstract Base Class）是一个具有纯虚函数的类，它用于定义接口或协议，但不能被实例化为对象

2.纯虚函数是在基类中声明的没有具体实现的虚函数。抽象基类提供了一个公共接口，而派生类需要实现这个接口中的纯虚函数，以完成具体的操作。在纯虚函数中，赋值为0是告诉编译器该函数是纯虚函数，从而让派生类实现自己的对应函数

用法如下：
1.定义接口：抽象基类用于定义一个通用的接口，其中包含一组纯虚函数。这些纯虚函数没有具体的实现，它们只是作为接口的声明存在，要求派生类提供具体的实现。通过定义接口，可以确保派生类都具备相同的行为和功能。
2.强制派生类实现：通过在抽象基类中声明纯虚函数，可以强制要求派生类提供这些函数的具体实现。这确保了派生类都有相同的接口，并且在编译时会进行静态类型检查，以确保派生类实现了所有纯虚函数。
3.多态性和运行时绑定：通过抽象基类和纯虚函数，可以实现多态性和运行时绑定的概念。通过基类指针或引用，可以处理不同类型的对象，而无需知道对象的具体类型。在运行时，根据指针或引用指向的对象的实际类型，调用相应的派生类实现。

虚继承

1.虚继承通过派生类继承时使用virtual关键字声明，如class cat: virtual public animal。用于解决多继承中的菱形继承问题（Diamond Inheritance Problem）和冗余继承问题（Redundant Inheritance Problem）。

2.在多继承中，如果派生类从两个或多个基类继承，并且这些基类又有一个共同的基类，就会产生菱形继承问题。这意味着派生类通过两条不同的路径继承了相同的基类，可能导致数据冗余和二义性。

3.虚继承通过引入虚基类（Virtual Base Class）来解决菱形继承问题。在虚继承中，通过在派生类的基类列表中使用 virtual 关键字来声明虚继承。被声明为虚基类的基类在继承体系中只存在一份共享的实例（即只有一个实例，避免在继承时生成过多的实例）。

虚继承具有以下特点和用途：
1.单一共享实例：虚继承确保虚基类在继承体系中只有一份实例，而不是被继承多次。
2.解决冗余和二义性：通过虚继承，避免了派生类中由于多次继承而导致的数据冗余和函数二义性的问题。
3.虚基类初始化：虚基类的构造函数由最派生类负责调用，确保虚基类的构造函数只被调用一次。
4.虚基类成员访问：通过派生类的对象可以访问虚基类的成员，但需要明确指定虚基类的名称。

4.限定符：

1.override限定符：通过使用override限定符来核实被覆盖的函数在基类中是否被声明为虚函数，且基类中相应虚函数的特征标是否与派生类中被声明为 override 的函数完全相同
2.final限定符：被声明为final的类不能作为基类，被声明为final的虚函数不能在派生类中覆盖
如：void Swim() override final
