std::map和std::multimap

map 和 multimap 是键-值对容器，支持高效的查找、插入和删除操作。它们之间的主要区别在于键的唯一性和元素排序方式，multimap可以存储重复的键，map只能存储唯一的键。使用时要包含头文件< map >

关联容器 std::map 和 std::multimap 内部使用红黑树这样的有序数据结构来存储元素。当我们向这些容器插入元素时，容器会根据元素的键值来将元素放置在合适的位置，以保持容器中元素的有序性。这样在遍历容器时，元素的键值会按照升序排列。

在向 std::map 或 std::multimap 插入元素时，键值是用于在红黑树中进行排序的关键。如果要替换容器中特定位置的元素，必须使用与现有元素相同的键值。如果插入一个新元素且其键值与容器中现有元素不同，容器会将该新元素放置在适当的位置，而不是直接替换原有元素。这是因为容器需要维护元素的有序性，新元素的键值可能导致位置上的排序发生变化。如果要实现替换元素的功能，可以先删除旧元素，然后插入新元素。

基于散列表的 STL 键-值对容器

从 C++11 起，STL 支持散列映射—std::unordered_map 类。要使用这个模板类，需要包含头文件< unordered_map >

unordered_map 的平均插入和删除时间是固定的，查找元素的时间也是固定的。

散列表的工作原理

散列表（Hash Table）是一种用于存储和查找数据的数据结构，其工作原理是通过将数据的键（Key）映射到存储位置的索引（Hash Value）来实现高效的查找和插入操作。可将散列表视为一个键-值对集合，根据给定的键，可找到相应的值。散列表与简单映射的区别在于，散列表将键-值对存储在桶中，每个桶都有索引，指出了它在散列表中的相对位置（类似于数组）。
这种索引是使用散列函数根据键计算得到的：Index = HashFunction(key, TableSize);使用 find( )根据键查找元素时，将使用 HashFunction( )计算元素的位置，并返回该位置的值，就像数组返回其存储的元素那样，具体工作原理如下：

  1. 哈希函数（Hash Function）：哈希函数是散列表的关键部分，它将数据的键映射到一个唯一的整数值，即哈希值。哈希函数的设计目标是使得不同的键产生不同的哈希值，尽可能避免不同键映射到同一个哈希值，这就是所谓的“哈希冲突”。
  2. 存储位置：将哈希值作为索引，将键值对存储在数组（或称为桶）的相应位置。这个数组称为散列表。通过哈希函数，我们可以在常数时间内找到数据的存储位置，从而实现快速的插入和查找。
  3. 哈希冲突处理：由于不同的键可能映射到相同的哈希值，可能会导致哈希冲突。为了处理哈希冲突，散列表使用了一些冲突解决方法，其中常见的有：
  4. 链接法（Chaining）：使用链表或其他数据结构将具有相同哈希值的键值对链接在一起。
    开放地址法（Open Addressing）：当发生冲突时，继续寻找下一个可用的存储位置，直到找到一个空位置。
