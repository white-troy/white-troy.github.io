类型转换运算符

1.类型转换是一种能够改变程序中对对象类型解释的机制。在这个机制中的操作符为类型转换运算符，它可以将对象转换为某个指定的类型。类型转换运算符有两种形式，一种是隐式转换运算符，一种是显式转换运算符。

2.隐式转换运算符允许在需要目标类型的表达式中隐式地自动执行类型转换。编译器会自动调用适当的类型转换运算符来执行隐式类型转换。在隐式转换中，要满足一个基本原则，即从低精度向高精度转换，如int a,double b,在执行a+b时，a会转变为double类型

在显式转换中，常用的有static_cast, dynamic_cast, reinterpret_cast, 和 const_cast 这四个转换运算符，这几个类型的转换运算符的使用语法相同：
destination_type result = cast_operator (object_to_cast);

3.1 static_cast(静态类型转换)

static_cast在编译时进行类的转换，可用于相关类型的指针之间进行转换，还可显式地执行标准数据类型的类型转换。在用于指针时，static_cast 实现了基本的编译阶段检查，确保指针被转换为相关类型。用于指针时可执行向上转换（指针转换为基类）也可执行向下转换，但向下转换时必须显示的调用，否则在运行时可能会出错。

reinterpret_cast用于不同类型之间的位模式转换，与转换对象的类型是否相关毫无关系。也就是说，这种转换是使程序进行强制重新解释类型。在这个过程中，他并不进行类型的检查，仅进行简单的位模式转换。

3.4 const_cast

const_cast用于去除常量性，可以将常量指针或引用转换为非常量的指针或引用，但不能用于修改本身是常量的对象（修改本身是常量的对象的意思就是，不能修改对象状态。如函数ShowAge()应该是只读的，在调用的时候展示年龄age的值，而不能修改age的大小）。

它主要用于实现函数重载时消除参数的常量性（常量对象是指在创建后不能被修改的对象，这就是常量性，而非常量对象是可以被修改的对象），或在某些情况下需要修改非常量对象的情况。

注意！除非万不得已，否则不要使用 const_cast 来调用非 const 函数。一般而言，使用 const_cast 来修改 const 对象可能导致不可预料的行为。
